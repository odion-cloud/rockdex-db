What is Rockdex DB?

Rockdex DB is a lightweight, feature-rich JavaScript database that works seamlessly in both Node.js and Browser environments. Perfect for building applications that need simple, fast data storage without complex setup.

üìö Quick Links:
‚≠ê View on GitHub üì¶ npm Package ‚Ä¢ npm install @odion-cloud/rockdex-db
üöÄ

Cross-Platform

Works perfectly in Node.js and browsers without any dependencies or platform-specific code.

üìÅ

Manual File Control

You create and manage .rdb files manually. Complete control over your database structure.

‚ö°

High Performance

In-memory operations with optional persistence. Fast queries with advanced WHERE conditions.

üîß

Feature Rich

Schema validation, transactions, triggers, joins, aggregations, and much more built-in.

‚ú® Zero Dependencies! Rockdex DB is a single file with no external dependencies. Works everywhere JavaScript runs.
üéØ Simple Philosophy: You control your data. Create files manually, handle encryption yourself, and manage storage exactly as you need.

Quick Example

// Create a database instance
const RockdexDB = require('@odion-cloud/rockdex-db');
const db = new RockdexDB({
    storageMode: 'memory',
    logging: true
});

// Create table and insert data
db.createTable('users');
db.insert('users', {
    name: 'John Doe',
    email: 'john@example.com',
    age: 30
});

// Query with advanced conditions
const adults = db
    .whereOperator('age', '>=', 18)
    .orderBy('name')
    .get('users');

console.log('Adult users:', adults);

Installation & Setup

1Install RockdexDB

npm install @odion-cloud/rockdex-db

2Import and Create Database

Import RockdexDB and create your first database instance:

// Node.js / CommonJS
const RockdexDB = require('@odion-cloud/rockdex-db');

// ES6 Modules
import RockdexDB from '@odion-cloud/rockdex-db';

// Create database instance
const db = new RockdexDB({
    storageMode: 'memory',  // or 'file'
    logging: true
});

3File Mode Setup (Optional)

For persistent storage, create your database structure manually:

# Create database folder
mkdir ./database

# Create table files
touch ./database/users.rdb
touch ./database/posts.rdb
touch ./database/orders.rdb

Then configure RockdexDB to use these files:

const db = new RockdexDB({
    storageMode: 'file',
    storagePath: './database',
    storageTable: ['users.rdb', 'posts.rdb', 'orders.rdb']
});

await db.ready(); // Wait for initialization

4Browser Usage

For browser usage, include the minified version:

<script src="node_modules/@odion-cloud/rockdex-db/rockdex-db.min.js"></script>
<script>
    const db = new RockdexDB({
        storageMode: 'memory',
        logging: true
    });
</script>
üéâ That's it! No complex configuration, no dependencies to install. RockdexDB is ready to use immediately.
üí° Pro Tip: Start with memory mode for testing, then switch to file mode for persistence when you're ready.

Quick Start Guide

1. Basic Operations

const RockdexDB = require('@odion-cloud/rockdex-db');
const db = new RockdexDB({ storageMode: 'memory', logging: true });

// Create table
db.createTable('users');

// Insert data
db.insert('users', {
    name: 'Alice Smith',
    email: 'alice@example.com',
    age: 28
});

// Query data
const users = db.get('users');
console.log(users);

2. Advanced Queries

// Complex WHERE conditions
const adults = db
    .whereOperator('age', '>=', 18)
    .whereLike('email', '%@gmail.com')
    .orderBy('name', 'ASC')
    .limit(10)
    .get('users');

// Aggregations
const avgAge = db.avg('users', 'age');
const userCount = db.count('users');

// Search across multiple fields
const results = db.search({
    name: 'Alice',
    email: 'example.com'
}).get('users');

3. Schema Validation

const userSchema = {
    name: { type: 'string', required: true },
    email: { type: 'string', required: true, pattern: /@/ },
    age: { type: 'number', min: 0, max: 120 }
};

db.createTable('users', userSchema);

// This will validate automatically
db.insert('users', {
    name: 'Bob Wilson',
    email: 'bob@example.com',
    age: 30
});

4. File Storage Mode

const fileDb = new RockdexDB({
    storageMode: 'file',
    storagePath: './database',
    storageTable: ['users.rdb', 'posts.rdb']
});

await fileDb.ready();

// Export data to file
const exportData = fileDb.getTableExport('users');
// Save manually: fs.writeFileSync('./backup.rdb', exportData);

// Import data from file
const fileContent = fs.readFileSync('./backup.rdb', 'utf8');
fileDb.importTable('users', fileContent);
üí° Remember: In file mode, you create the .rdb files manually. RockdexDB provides export/import functionality but doesn't automatically write files.

Real-World Examples

üõí E-commerce Product Catalog

const db = new RockdexDB({ storageMode: 'memory' });

// Create products table with schema
const productSchema = {
    name: { type: 'string', required: true },
    price: { type: 'number', required: true, min: 0 },
    category: { type: 'string', required: true },
    inStock: { type: 'boolean', required: false }

    { name: 'Coffee Mug', price: 15, category: 'Kitchen', inStock: true },

];

db.bulkInsert('products', products);

// Find products under $100

    .whereOperator('price', '<', 100)
    .where('inStock', true)
    .orderBy('price', 'ASC')
    .get('products');

console.log('Affordable products:', affordableProducts);

üîÑ Transaction Example

const db = new RockdexDB({ storageMode: 'memory' });

db.createTable('accounts');
db.createTable('transactions');

// Create initial accounts
db.insert('accounts', { name: 'Alice', balance: 1000 });
db.insert('accounts', { name: 'Bob', balance: 500 });

// Transfer money between accounts
try {
    db.transaction(db => {
        // Find accounts
        const alice = db.where('name', 'Alice').getOne('accounts');
        const bob = db.where('name', 'Bob').getOne('accounts');
        
        const transferAmount = 200;
        
        // Check sufficient funds
        if (alice.balance < transferAmount) {
            throw new Error('Insufficient funds');
        }
        
        // Update balances
        db.where('id', alice.id).update('accounts', { 
            balance: alice.balance - transferAmount 
        });
        db.where('id', bob.id).update('accounts', { 
            balance: bob.balance + transferAmount 
        });
        
        // Log transaction
        db.insert('transactions', {
            from: alice.id,
            to: bob.id,
            amount: transferAmount,
            type: 'transfer'
        });
    });
    
    console.log('Transfer completed successfully');
} catch (error) {
    console.error('Transfer failed:', error.message);
}

Complete API Reference

üèóÔ∏è Configuration Options

const db = new RockdexDB({
    storageMode: 'memory' | 'file',     // Storage type
    storagePath: './data',              // Path for file mode
    storageTable: ['table1.rdb'],       // Table files for file mode
    logging: true | false | function,   // Enable logging
    timestamps: true | false,           // Auto-add created_at/updated_at
    softDelete: true | false,           // Use soft deletes
    defaultData: {                      // Initial data for memory mode
        users: [{ name: 'Admin' }]
    }
});

üìã Core Methods

createTable(tableName, schema?)

Create a new table with optional schema validation.

db.createTable('users', {
    name: { type: 'string', required: true },
    email: { type: 'string', pattern: /@/ },
    age: { type: 'number', min: 0, max: 120 }
});
insert(tableName, data)

Insert a single record into the specified table.

db.insert('users', {
    name: 'John Doe',
    email: 'john@example.com',
    age: 30
});
where(field, value, operator?)

Add WHERE condition to query. Chain multiple conditions.

// Simple equality
db.where('status', 'active').get('users');

// With custom operator
db.where('age', 18, 'AND').whereOperator('age', '>=', 18);
get(tableName)

Execute query and return all matching records.

const users = db
    .where('active', true)
    .orderBy('name')
    .limit(10)
    .get('users');

üîç Query Methods

Method Description Example
whereOperator(field, op, value) WHERE with custom operator db.whereOperator('age', '>', 18)
whereIn(field, values) WHERE field IN array db.whereIn('status', ['active', 'pending'])
whereLike(field, pattern) WHERE field LIKE pattern db.whereLike('name', 'John%')
orderBy(column, direction) Sort results db.orderBy('created_at', 'DESC')
limit(count, offset?) Limit and pagination db.limit(10, 20)
search(conditions) Multi-field search db.search({name: 'John', email: 'gmail'})

üìä Aggregation Methods

count(tableName)

Count matching records

db.where('active', true).count('users')

sum(tableName, column)

Sum numeric column

db.sum('orders', 'amount')

avg(tableName, column)

Average of numeric column

db.avg('products', 'price')

min/max(tableName, column)

Minimum/maximum value

db.max('users', 'age')

üîÑ Advanced Features

transaction(callback)

Execute multiple operations atomically.

db.transaction(db => {
    db.insert('users', userData);
    db.insert('profiles', profileData);
    // If any operation fails, all changes are rolled back
});
createTrigger(tableName, triggerName, callback)

Create event-driven database operations.

db.createTrigger('users', 'beforeInsert', ({ NEW }) => {
    NEW.created_at = new Date().toISOString();
    return true; // Allow operation
});
join(table1, table2, key1, key2)

Join two tables on specified keys.

const usersWithProfiles = db.join('users', 'profiles', 'id', 'userId');
